# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any Citizen Weather Foundation project, please **do not** open a public GitHub issue.

Instead, please email security concerns to the project maintainers with:
- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggestions for patching

We take security seriously and will acknowledge your report within 48 hours and provide a more detailed response within 5 business days.

## Security Practices

We follow these practices to maintain the security of our projects:

- **Dependency Management**: We regularly update dependencies and monitor for known vulnerabilities
- **Code Review**: All code contributions go through review before merging
- **Access Control**: Repository access is limited to authorized maintainers
- **Data Privacy**: We respect user privacy and follow data protection best practices

## Supported Versions

Security updates are provided for:
- The latest stable release
- The previous major version for 6 months

Users are encouraged to upgrade to the latest version to receive security updates.

## Responsible Disclosure

We appreciate researchers and security professionals who responsibly disclose security issues. We commit to:
- Acknowledging receipt of your report
- Providing updates as we work on a fix
- Crediting you for the discovery (if desired)
- Allowing reasonable time for fixes before public disclosure

Thank you for helping keep Citizen Weather Foundation secure!
