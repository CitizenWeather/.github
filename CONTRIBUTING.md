# Contributing to CitizenWeather

Thank you for your interest in contributing to CitizenWeather! We welcome contributions from everyone in the community.

## 🤝 Our Community

CitizenWeather is built by our community. We're committed to creating an inclusive and welcoming environment for all contributors.

## 📖 Code of Conduct

Please review our [Code of Conduct](./CODE_OF_CONDUCT.md) before participating. All contributors are expected to uphold this code.

## 🎯 Ways to Contribute

- **Report Bugs** — Help us identify and fix issues
- **Suggest Features** — Share ideas for improvements
- **Improve Documentation** — Help make docs clearer and more comprehensive
- **Write Code** — Submit pull requests with bug fixes or new features
- **Review Code** — Help review pull requests from other contributors
- **Join Discussions** — Participate in community conversations

## 🐛 Reporting Bugs

Before creating a bug report, please check existing issues to avoid duplicates.

When creating a bug report:

1. Use the bug report issue template
2. Include a clear, descriptive title
3. Provide a detailed description of the issue
4. Include steps to reproduce the problem
5. Describe the observed and expected behavior
6. Add any relevant logs, screenshots, or error messages
7. Specify your environment (OS, version, etc.)

## 💡 Suggesting Features

We love hearing ideas for new features!

When suggesting a feature:

1. Use the feature request issue template
2. Provide a clear, descriptive title
3. Explain the use case and expected benefits
4. Describe the desired behavior
5. Consider implementation challenges
6. Check if similar features already exist

## 💻 Pull Request Process

### Before You Start

1. **Create or Find an Issue** — Discussions should happen on GitHub issues first
2. **Fork the Repository** — Create your own fork to work on
3. **Create a Feature Branch** — Use a descriptive branch name (e.g., `fix/issue-123` or `feat/new-feature`)

### Making Changes

1. **Follow Code Standards** — Maintain consistency with the project's style and conventions
2. **Write Tests** — Add tests for new features and bug fixes
3. **Update Documentation** — Keep docs in sync with code changes
4. **Keep Commits Clean** — Use clear, descriptive commit messages
5. **Sign Your Commits** — If required, sign commits with GPG

### Submitting a Pull Request

1. **Use the PR Template** — Follow the pull request template guidelines
2. **Link Related Issues** — Reference issues with `Closes #123` or `Fixes #123`
3. **Describe Your Changes** — Clearly explain what and why you changed
4. **Request Reviews** — Ask relevant maintainers or teams to review
5. **Respond to Feedback** — Be open to suggestions and iterate
6. **Keep It Focused** — Each PR should address a single concern

### PR Checklist

- [ ] Code follows project style guidelines
- [ ] Tests are included and passing
- [ ] Documentation is updated
- [ ] Commit messages are clear and descriptive
- [ ] No unrelated changes are included
- [ ] Related issues are linked

## 📋 Development Setup

Refer to your project's README for specific setup instructions. Generally:

1. Clone the repository
2. Install dependencies
3. Set up development environment
4. Run tests locally
5. Create your feature branch

## ✅ Review Process

All pull requests go through code review:

- **Automated Checks** — Tests, linters, and security scans must pass
- **Code Review** — At least one maintainer will review your changes
- **Feedback** — Address any feedback or suggestions
- **Approval** — Once approved, your PR can be merged

## 📦 After Merge

- Your contribution becomes part of the codebase
- You'll be credited in release notes (if applicable)
- Celebrate your contribution! 🎉

## ⚖️ Legal

By contributing to CitizenWeather, you agree that your contributions will be released under the project's license. You confirm that you have the right to contribute and that your contributions don't violate any laws or third-party rights.
