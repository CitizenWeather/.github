# Contributing to Citizen Weather Foundation

Thank you for your interest in contributing! We welcome contributions from everyone, whether you're a weather enthusiast, developer, data scientist, or just someone passionate about open data.

## Getting Started

1. **Review the Code of Conduct**: Please read our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) to understand our community standards.

2. **Check Existing Issues**: Look at open issues and discussions to see if someone is already working on what you want to do.

3. **Start Small**: First-time contributors should look for issues tagged with `good-first-issue` or `help-wanted`.

## How to Contribute

### Reporting Bugs

Before creating a bug report, please:
- Check the issue tracker to see if the bug has already been reported
- Provide a clear description of what the bug is
- Include steps to reproduce the issue
- Describe the observed behavior and what you expected to happen
- Include screenshots or logs if applicable

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion:
- Use a clear, descriptive title
- Provide a step-by-step description of the suggested enhancement
- Explain why this enhancement would be useful
- List some examples of projects or tools that have similar functionality

### Making Code Contributions

1. **Fork and Create a Branch**
   ```bash
   git clone https://github.com/citizenweather/your-repo.git
   git checkout -b feature/your-feature-name
   ```

2. **Make Your Changes**
   - Follow the project's coding style and conventions
   - Write clear commit messages
   - Keep commits focused and atomic

3. **Test Your Changes**
   - Run any existing tests
   - Add new tests for new functionality
   - Ensure your code doesn't break existing features

4. **Submit a Pull Request**
   - Provide a clear description of what your PR does
   - Reference any related issues
   - Be responsive to feedback and review comments

## Development Setup

Each project has its own setup instructions in their respective repository READMEs. Check the specific project you're contributing to for:
- Required dependencies
- Development environment setup
- How to run tests
- Code style guidelines

## Questions?

- 💬 Start a [GitHub Discussion](https://github.com/orgs/citizenweather/discussions)
- 📧 Reach out to the maintainers in the project's documentation
- 🐦 Check our community channels

Thank you for helping us build weather data for everyone! 🌤️
